## 📋 Project Overview

An end-to-end AI system that helps HR departments predict employee attrition, forecast performance ratings, and segment employees into actionable clusters — all powered by machine learning and deployed on Databricks.

## 🔄 Pipeline Phases

### Phase 1 — Data Ingestion & Preprocessing
- Loaded 1,470 rows × 35 columns
- Removed 3 useless columns (EmployeeCount, Over18, StandardHours)
- Label Encoding: Attrition, Gender, OverTime
- One-Hot Encoding: BusinessTravel, Department, EducationField, JobRole, MaritalStatus
- StandardScaler applied to 26 numerical columns
- 80/20 Train/Test split with stratification
- SMOTE applied to balance classes: 986 vs 986 (from 986 vs 190)

**Output files:**
```
Data/processed/hr_cleaned.csv
Data/processed/X_train_balanced.csv
Data/processed/y_train_balanced.csv
Data/processed/X_test.csv
Data/processed/y_test.csv
```

## 📌 Important Notes

- SMOTE was applied **only on training data** — test data is never touched
- Use **F1-Score** (not accuracy) to evaluate attrition model due to class imbalance
- All file paths are relative — run from inside the `cloudDB_project/` folder
- MLflow warnings about pickle format are expected and safe for academic use
